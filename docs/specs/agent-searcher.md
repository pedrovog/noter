# Searcher Agent

Collects raw content from sources. Operates in two parallel tracks and checks the cache before any external request.

## Inputs

- `queries: [str]` — queries generated by the Planner
- `user_urls: [str]` — URLs provided via `--source` and `--source-file`
- `n_sources: int` — limit of automatic sources (default: 5)
- `cache_ttl: int` — cache TTL in days (default: 30)
- `no_cache: bool`

## Output

```python
[
  {
    "url": str,
    "title": str,
    "content": str,         # scraped markdown
    "source": "auto|user",  # origin
    "from_cache": bool
  }
]
```

## Track A — Automatic Search

1. Firecrawl search with each query from the Planner
2. arXiv API — search by term, returns abstracts + links
3. Wikipedia API — main article for the topic if it exists
4. Respects `n_sources` across all three sub-sources combined
5. Checks `scrape_cache` before scraping each URL

## Track B — User URLs

1. Checks cache before scraping
2. Scraping via Firecrawl scrape if not cached
3. No limit — all user URLs are always processed
4. Displays warning if URL was already used in another note (via `url_usage`)

## Parallelism

Tracks A and B run in parallel with `asyncio` or `ThreadPoolExecutor`. Within Track A, multiple queries also run in parallel.
