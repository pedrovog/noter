# Linker Agent

Enriches generated notes with `[[links]]` to existing notes in the vault.

## Inputs

- `note_paths: [str]` — paths of files generated by the Writer
- `vault_path: str` — vault root for indexing existing titles

## Output

`.md` files updated in-place with `[[links]]` injected.

## Logic

### 1. Vault Indexing

Reads all `.md` files in the vault and extracts titles (`# Title` heading or filename). Builds a `title → path` dictionary.

### 2. Link Detection

Sends the note content + vault title list to Claude. Claude identifies where in the text a vault title is mentioned (or strongly implied) and returns substitutions: `{original_text, text_with_link}`.

### 3. Injection

Applies substitutions to the file. Adds linked titles to the `## Connections` section.

## Rules

- Do not link to the note itself (no self-reference)
- Do not link the same title more than once in the body (first occurrence only)
- Do not modify frontmatter
- If no links found: leave `## Connections` empty, no error

## Claude Tools

No tool use — structured response via JSON mode.
